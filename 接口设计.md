# Excel 智能翻译云函数 API 接口设计文档

## 概述
本项目后端基于腾讯云 Serverless 云函数 (SCF) 构建，通过 API 网关提供 HTTP 访问。共包含三个核心接口：
1. **认证接口 (`auth-function`)**：负责卡密校验和额度查询。
2. **解析接口 (`parse-function`)**：负责读取前端上传的 Excel 文件，提取需要翻译的单元格。
3. **翻译接口 (`translate-function`)**：负责调用大模型进行翻译，将结果写回 Excel，并处理额度扣除逻辑。

### 全局说明
- **通信协议**：HTTPS
- **请求方法**：主要为 `POST`，所有接口均支持 `OPTIONS` 预检请求以处理 CORS 跨域。
- **请求头 (Headers)**：
  - `Content-Type: application/json`
- **基础 URL (Base URL)**：`https://wanxin1994-3g0nf07m2ee74437.api.tcloudbasegateway.com/v1/functions`

## 1. 卡密校验接口 (Auth API)

**路径**: `/auth-function`
**方法**: `POST`
**描述**: 验证用户输入的卡密是否有效，并返回关联的用户 ID 和剩余翻译额度（单元格数量）。

### 请求参数 (Body)
| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `key` | String | 是 | 用户输入的授权卡密 |

**请求示例**:
```json
{
  "key": "test_card_key_12345"
}
```

### 响应数据
**成功响应 (200 OK)**:
```json
{
  "valid": true,
  "user_id": "user_001",
  "remaining_count": 1000
}
```

**失败响应 (400 / 200)**:
```json
{
  "valid": false,
  "error": "无效的卡密" // 或 "卡密不能为空"
}
```

## 2. Excel 解析接口 (Parse API)

**路径**: `/parse-function`
**方法**: `POST`
**描述**: 接收前端上传的 Excel 文件的 Base64 编码，解析第一个工作表 (Sheet)，提取所有非空单元格的坐标和文本内容。

### 请求参数 (Body)
| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `file_base64` | String | 是 | 包含 MIME 头或纯 Base64 编码的 Excel 文件数据 |

**请求示例**:
```json
{
  "file_base64": "data:application/vnd.openxmlformats-officedocument.spreadsheetml.sheet;base64,UEsDBBQABgA..."
}
```

### 响应数据
**成功响应 (200 OK)**:
```json
{
  "preview": [
    {
      "coordinate": "A1",
      "text": "Hello"
    },
    {
      "coordinate": "B1",
      "text": "World"
    }
  ],
  "total_cells": 2,
  "sheet_name": "Sheet1"
}
```

**失败响应 (400 / 500)**:
```json
{
  "error": "未接收到文件数据" // 或解析过程中的异常信息
}
```

## 3. 翻译与处理接口 (Translate API)

**路径**: `/translate-function`
**方法**: `POST`
**描述**: 核心业务接口。执行以下流程：
1. 校验用户额度是否足够。
2. 调用 ModelScope 平台的大模型 (Qwen) 翻译传入的数据字典。
3. 将翻译结果写回原 Excel 文件的对应坐标中。
4. 在数据库中扣除对应的翻译单元格额度，并记录使用日志。
5. 将处理后的新 Excel 文件以 Base64 格式返回给前端下载。

### 请求参数 (Body)
| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `user_id` | String | 是 | 从 Auth 接口获取的用户 ID |
| `target_lang` | String | 是 | 目标语言，例如 "中文", "English" |
| `file_base64` | String | 是 | 原始 Excel 文件的 Base64 编码 |
| `sheet_name` | String | 否 | 需要操作的工作表名称，默认取第一个表 |
| `data` | Object | 是 | 待翻译的单元格键值对，格式为 `{"坐标": "原文"}` |

**请求示例**:
```json
{
  "user_id": "user_001",
  "target_lang": "中文",
  "file_base64": "data:application/vnd.openxml...base64...",
  "sheet_name": "Sheet1",
  "data": {
    "A1": "Hello",
    "B1": "World"
  }
}
```

### 响应数据
**成功响应 (200 OK)**:
```json
{
  "file_url": "data:application/vnd.openxmlformats-officedocument.spreadsheetml.sheet;base64,UEsDBBQABgA...",
  "used_count": 2
}
```

**失败响应 (400 / 403 / 500)**:
```json
{
  "error": "卡密剩余次数不足" // 或 "未找到该用户的授权卡密", "缺少必要参数", "翻译处理失败: xxx"
}
```
