# 云函数接口开发通用规范

## 1. 概述
本规范旨在统一云函数的开发风格，确保接口的入参解析、业务逻辑处理、异常捕获及响应格式的一致性。所有云函数应遵循此规范进行开发和维护。

## 2. 依赖与初始化
每个云函数必须引入 `@cloudbase/node-sdk` 并初始化应用实例，以确保能正确访问云数据库和其他云资源。

```javascript
const cloudbase = require('@cloudbase/node-sdk');

const app = cloudbase.init({
  env: "wanxin1994-3g0nf07m2ee74437" // 替换为实际的环境 ID
});
const db = app.database();
const _ = db.command;
```

## 3. 请求处理规范

### 3.1 入参解析
云函数通过 `event` 对象接收请求参数。
- **直接使用 `event` 对象**：API 网关配置为“集成响应”或特定触发模式时，请求体通常已自动解析并合并到 `event` 中。
- **不再手动解析 `event.body`**：除非明确知道请求是 Base64 编码或原始字符串，否则直接从 `event` 中读取字段。

**示例代码**:
```javascript
exports.main = async (event, context) => {
  // 直接从 event 中解构参数
  const { key, file_base64, user_id } = event;
  
  // 参数校验
  if (!key) {
    // ... 返回错误
  }
};
```

### 3.2 跨域处理 (CORS)
虽然业务逻辑中不再强制要求手动处理 OPTIONS 请求（通常由网关层处理），但为了兼容性，建议保留 `Access-Control-Allow-Origin` 等响应头，或者直接在返回结构中不包含 headers，由网关统一配置。
*注：当前 auth-function 示例中未包含 headers 返回，说明网关层可能已配置或使用默认行为。建议统一返回格式如下。*

## 4. 响应格式规范 (Response)

所有接口必须返回一个包含 `isBase64Encoded`, `statusCode`, `data` (或 `body`) 的对象。
**特别注意**：为了适配当前的网关配置，成功的业务数据直接放在 `data` 字段中（JSON 对象），而不是序列化后的 `body` 字符串。

### 4.1 成功响应
- **isBase64Encoded**: `false`
- **statusCode**: `200`
- **data**: 业务数据对象 (JSON Object)

```javascript
return {
  isBase64Encoded: false,
  statusCode: 200,
  data: {
    valid: true,
    user_id: "user_123",
    // 其他业务字段...
  }
};
```

### 4.2 业务错误响应 (逻辑错误)
当业务逻辑无法继续（如卡密无效、余额不足）时，返回 200 状态码，并在 data 中通过标志位体现错误。

```javascript
return {
  isBase64Encoded: false,
  statusCode: 200, // 业务层面错误通常仍返回 200 HTTP 状态码
  data: {
    valid: false,
    error: "无效的卡密"
  }
};
```

### 4.3 系统错误响应 (异常)
当发生代码异常或系统级错误时，捕获异常并返回 500。

```javascript
return {
  isBase64Encoded: false,
  statusCode: 500,
  data: {
    valid: false, // 可选
    error: `Server Error: ${e.message}`
  }
};
```

### 4.4 参数错误响应
当缺少必填参数时，返回 400。

```javascript
return {
  isBase64Encoded: false,
  statusCode: 400,
  data: {
    valid: false,
    error: "参数不能为空"
  }
};
```

## 5. 代码模板

以下是一个标准的云函数代码模板，适用于新建接口：

```javascript
const cloudbase = require('@cloudbase/node-sdk');

// 初始化
const app = cloudbase.init({
  env: "wanxin1994-3g0nf07m2ee74437"
});
const db = app.database();

exports.main = async (event, context) => {
  console.log('Function invoked with event:', event);

  try {
    // 1. 参数提取
    const { param1, param2 } = event;

    // 2. 参数校验
    if (!param1) {
      return {
        isBase64Encoded: false,
        statusCode: 400,
        data: { error: '缺少必要参数 param1' }
      };
    }

    // 3. 业务逻辑 (例如查询数据库)
    // const res = await db.collection("collection_name").where({ ... }).get();

    // 4. 成功返回
    return {
      isBase64Encoded: false,
      statusCode: 200,
      data: {
        success: true,
        result: "业务处理结果"
      }
    };

  } catch (e) {
    console.error('System Error:', e);
    // 5. 异常返回
    return {
      isBase64Encoded: false,
      statusCode: 500,
      data: { error: `Server Error: ${e.message}` }
    };
  }
};
```

## 6. 关键注意事项
1. **日志记录**：使用 `console.log` 打印关键步骤（如数据库查询结果），便于在云端日志中排查问题。
2. **环境 ID**：确保 `cloudbase.init` 中的 `env` ID 与当前部署环境一致。
3. **数据库操作**：使用 `await` 异步等待数据库操作完成。
4. **依赖管理**：新引入第三方库时，务必在 `package.json` 中声明并在本地测试通过后再打包上传。
